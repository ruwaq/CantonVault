# CantonVault UX Redesign — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the contrast bug (illegible text in dark mode), humanize ~30 jargon strings, refactor the 898-line `VaultView.tsx` monolith into focused components, and ship a 4-screen propose wizard — all without touching the working backend/logic layer.

**Architecture:** Presentation-only redesign on top of an intact SWR + hooks + stores + Pages Functions stack. New 3-tier design-token system (primitive→semantic→component) fixes contrast at the root. A `lib/copy.ts` dictionary centralizes human language. Components are extracted from the monolith into `components/vault/*`, each <200 lines. The 7-field propose form becomes a 4-screen wizard (1 decision per screen).

**Tech Stack:** React 18 + TypeScript + Vite + Bootstrap 5 + SWR. CSS custom properties (no CSS-in-JS). Cloudflare Pages deployment via `git push` (auto-deploy).

**Source spec:** `docs/superpowers/specs/2026-07-14-ux-redesign-design.md` (667 lines, approved).

**Decisions confirmed this session:**
- Language: **English plain** (Cash App vocabulary, no Daml jargon).
- Mockups: **None** — use the ASCII layouts in the spec, go straight to code.

**Build/verify commands:**
- Local typecheck+build: `cd cn-quickstart/quickstart/frontend && npm run build:ci`
- Deploy: `git push origin main` (auto-deploy to `canton-vault.pages.dev`)
- Health check: `curl -s https://canton-vault.pages.dev/api/health`
